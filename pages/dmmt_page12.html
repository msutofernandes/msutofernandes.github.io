Chapter 12 – Accessibility

When developing an accessible Web site, developers usually face two problems: more work and compromised design. But, it’s possible to make any site accessible by focusing on a few important things and without a lot of effort.

  1. Test your site very often, focusing on something that confuses most people (not only people with disabilities) and fix it.
  2. Most of us have no experience at using adaptive technologies. There is an article titled “Guidelines for Accessible and Usable Web Sites: Observing Users Who Work with Screen Readers.” They observed 16 blind users doing a number of tasks using a screen reader.
  3. Read these two books: A Web for Everyone: Designing Accessible User Experiences and Web Accessibility: Web Standards and Regulatory Compliance.
  4. Add a helpful and descriptive alt text to every image, use headings correctly, associate fields with their text labels using the tag label, make possible to skip to main content and all content accessible by keyboard, create a contrast between the text and the background and use an accessible template.
